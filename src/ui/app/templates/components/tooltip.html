It does NOT add a second tooltip engine: - static/js/main.js wires up every [data-bs-toggle="tooltip"] on DOMContentReady via `new bootstrap.Tooltip(el)`. - static/js/i18n.js re-syncs the visible bubble's text via bootstrap.Tooltip.getInstance(el).setContent(...) whenever the locale changes. - Pages that inject markup after load (plugins-settings.js, modules/setting_controls.js) already call `new bootstrap.Tooltip(el)` themselves on the new node -- do the same for anything you inject dynamically via this macro; nothing new to wire up. Attribute contract emitted: data-bs-toggle="tooltip" data-bs-original-title="{text}" -- never a plain `title`: a raw title flashes the browser's native tooltip before bootstrap.js runs. data-bs-placement="{placement}" data-bs-trigger="{trigger}" -- only emitted when it differs from the default. data-bs-html="true" -- only when html=true. data-bs-delay="{delay}" -- only when delay is set. data-bs-custom-class="warning-tooltip" -- only when variant="warning". Bootstrap 5 reads this data attribute and adds the class to the generated .tooltip popup itself (which is appended next to , not nested under the trigger, so a class on the trigger would never reach it). data-i18n="{i18n_key}" -- ONLY when html=false: i18n.js's applyTranslations() special-cases `[data-bs-original-title]` elements and rewrites that attribute directly on locale change (see navbar.html's "Version" link, or the reset-setting button in models/input_setting.html). When html=true the translatable text is markup the caller controls (see menu.html's "Pro feature" tooltip) -- embed your own inside `text` yourself in that case. Params: text (required) English fallback tooltip copy (or trusted HTML when html=true). i18n_key data-i18n key applied to the trigger, e.g. "tooltip.restart_scheduler". Ignored when html=true -- see contract above. placement "top" | "right" | "bottom" | "left". Default "top". trigger space-separated Bootstrap trigger events. Default "hover focus" -- keep "focus" in the set so keyboard users get the hint too. html render `text` as HTML (data-bs-html="true"). Trusted input only. Default false. variant none | "warning" -- the only existing color variant (.warning-tooltip, overrides.css, recolors the bubble to bg-warning). Don't invent another. disabled add tabindex="0" to the wrapper so the tooltip still opens when caller() is a disabled control -- a disabled element fires no pointer/focus events, so a tooltip attached to it directly never opens. Default false. delay optional ms before show (data-bs-delay). id optional id on the rendered wrapper element. tag wrapper element name.
