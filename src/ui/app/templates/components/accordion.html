Two macros, always used together: accordion(id, ...) the outer ".accordion" wrapper. Body via {% call %}, containing one or more accordion_item() calls. accordion_item(id, title, ...) one collapsible section. Body via {% call %} -- any markup or other component is fair game inside. Pass the SAME id you gave accordion() as `parent_id` on each accordion_item() to get single-open-at-a-time behaviour (opening one section closes its siblings, via Bootstrap's `data-bs-parent`). Omit `parent_id` on an item and it toggles independently of the others, any number open at once -- both behaviours coexist inside one accordion() if you mix it. Params -- accordion(): id required -- id on the outer .accordion div. Default false. classes extra utility classes on the outer wrapper. Params -- accordion_item(): id required -- unique id for the collapsible panel. Builds "{id}-header" for the heading/aria-labelledby link and is the data-bs-target the button points at. title required -- English fallback header text (the always-visible trigger copy). title_i18n data-i18n key for title. parent_id accordion() id to close sibling sections on open (data-bs-parent). Omit for a section that toggles independently of the rest of the group. open start expanded. Default false. icon optional leading Boxicons class, e.g. "bx-lock-alt". subtitle optional trailing caption before the chevron -- e.g. a SETTING name, rendered in the app's existing courier-prime monospace face (see components/secret-field.html) since that's what it's for here too. subtitle_i18n data-i18n key for subtitle. disabled render the trigger inert (disabled button, can't be toggled). Default false. heading heading tag wrapping the button, for correct document outline in whatever context the accordion is embedded ("h2" | "h3" | "h4" ...).
