{# usage: {% from "components/spinner.html" import spinner %} {{ spinner(size="sm", label="Loading reports...", label_i18n="status.loading_reports") }} #} {# Bare loading glyph -- a thin wrapper around Bootstrap's own ".spinner-border" (core.css, already vendored, size variants "sm"/""/"lg" all pre-existing -- no new CSS needed here). Two shapes, picked by whether `label` is passed: no label a standalone glyph, `role="status"` on the glyph itself with a ".visually-hidden" fallback caption for screen readers (the plain Bootstrap spinner pattern) -- for a spinner sitting next to/inside something that already has its own visible caption (a card header, a button -- see components/button.html's own `loading` param, which renders this exact shape inline and is deliberately left as-is rather than refactored onto this macro). label set an inline "spinner + visible text" row (`role="status" aria-live="polite"` on the outer wrapper, the glyph itself `aria-hidden` so the visible text is what gets announced, not a redundant one) -- the shape reports.js hand-rolls today for its "Loading report details..." modal placeholder: `Loading report details...`. Porting reports.js onto this macro is separate page-by-page work (Phase 3), not done here. Params: size 'sm' | 'md' | 'lg'. Maps to spinner-border-sm / spinner-border / spinner-border-lg (all three already in core.css). Default 'md'. variant color suffix appended to "text-", e.g. "primary", "success", "danger", "info", "secondary", "muted". Must be a color with an existing ".text-" rule. Default "primary". label optional visible caption rendered beside the glyph (English fallback). Omit for the bare-glyph shape. label_i18n data-i18n key for label. Optional -- omit for the rare untranslated case. id optional id on the outer/root element. classes extra utility classes on the outer/root element. Not covered: - A full loading PANEL (icon/title/message, its own layout) -- components/empty-state.html's state="loading" already owns that larger shape; this macro is only the glyph + optional inline caption. - A blocking overlay pinned over an existing card/table while it refreshes -- no such component exists yet in this codebase; add one separately if a page actually needs it rather than growing this macro to cover a shape nothing here uses today. - bans.js's / reports.js's ".bx-loader" + ".bx-spin" toggle on the "Auto refresh" button icon -- that's a persistent ON/OFF toggle indicator (Boxicons rotate), not a transient "waiting for a response" spinner, and isn't Bootstrap's `.spinner-border` at all. Different affordance, left untouched. No companion JS for the glyph itself -- inert markup, same as badge.html. static/js/components/spinner.js instead ships a separate, standalone utility, `setButtonLoading(button, isLoading)`, for toggling any
